# Contributing (Issues Only)

Thanks for helping improve RequestFlow.

## What we accept

- Bug reports
- Improvement proposals
- Usability feedback
- Documentation feedback

## What we do not accept

- External source code contributions
- External pull requests

## How to submit good feedback

1. Search existing issues first.
2. Use the issue templates.
3. Include clear reproduction steps for bugs.
4. Include version and environment details.
5. Add screenshots or logs when relevant.

## Process

- Maintainers triage issues with labels.
- Accepted items are planned in the internal engineering backlog.
- Major updates are shared through issue/discussion follow ups.
