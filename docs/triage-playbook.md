# Triage Playbook

## Daily triage

1. Label new issues with needs-triage.
2. Validate duplicate reports.
3. Ask for missing repro steps when needed.
4. Apply priority label.
5. Move accepted items to internal backlog.

## Suggested labels

- type: bug
- type: enhancement
- type: docs
- status: needs-triage
- status: needs-info
- status: accepted
- status: closed-not-planned
- priority: p1
- priority: p2
- priority: p3

## Response style

- Acknowledge quickly.
- Be transparent when no ETA is available.
- Close with reason when out of scope.
